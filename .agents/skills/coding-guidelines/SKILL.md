# Coding Guidelines

## Apply the guidelines

Read and apply [the coding guidelines](references/coding-guidelines.md) before implementing or reviewing code.

Treat language- or project-specific practices as additions to these guidelines, not replacements. When guidance conflicts, follow the project's explicit conventions and state the relevant tradeoff in a review.

For reviews, identify concrete deviations that affect correctness, maintainability, API behavior, error handling, ownership, or test coverage. Do not raise purely stylistic comments that the guidelines do not support.
